    1. File Viewing Commands

  • cat [opt] filename

    A command useful for viewing simple (short) files.
The -n option displays line numbers along with the content.

  • tail [opt] filename

    Displays the last part of the file.
Without options, it shows the last 10 lines by default.
-n option: shows the last n lines.
-f option: monitors the file in real-time for changes.

    2. File Manipulation Commands

  • cp [opt] original_filename copy_filename

    - cp file1 file2: Copies file1 to file2.
- cp -r dir1 dir2: Recursively copies dir1 and all contents to dir2.

  • rm [opt] filename/dirname

      option

    • Prompt before deleting

      -i option prompts before deletion.

    • Force delete

      Use -f to force delete, especially when the directory contains files.

    • Delete directory

      Use -r option to remove a directory and its contents.

Difference Between ln (link) and cp (copy)

ln -s/no opt original new_file creates a new file sharing the same inode and content.
Though it looks similar to cp, the differences are:

cp (copy)
- Creates a completely separate file.
- Modifying one does not affect the other.
- Used for editing separate copies.

ln (link)
- Shares content under different names.
- Modifying one affects the other.
- Used when files need to be managed jointly.

This difference arises because cp creates a new file with a different inode,
while ln creates a new file pointing to the same inode.

How to update file timestamp or create an empty new file without modifying it

touch: If the specified file doesnt exist, it creates it.
If it exists, it updates its timestamp to the current time or a specified time.
