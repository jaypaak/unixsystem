File Attributes

Unix is a multi-user system that provides file access permissions to ensure security. File attributes can be checked using the ls -l command. Attributes include file type, permissions, link count, owner, group, size, and last modified time.

File Access Permissions

Users are categorized into three groups, each with different permissions:

  • Owner (u): The user who created the file.
  • Group (g): Users who belong to the same group as the file.
  • Others (o): All other users.

Permissions include:

  • r: read
  • w: write
  • x: execute

Changing Permissions with Symbols

chmod [options] mode filename

Examples:

  • chmod u-w file.txt: Remove write permission from owner
  • chmod g+wx file.txt: Add write and execute to group
  • chmod o=rwx file.txt: Set full permissions for others

Changing Permissions with Numbers

Permissions can also be changed using octal numbers:

  • 7 = rwx
  • 6 = rw-
  • 5 = r-x
  • 4 = r--

Examples:

  • chmod 755 file.txt: Owner rwx, Group and Others r-x
  • chmod 700 file.txt: Owner rwx, Group and Others no access

Default File Permissions

When a file or directory is created, default permissions are applied, modified by a umask value.

  • umask: View or set the permission mask
  • Example: umask 022 results in default 755 for directories and 644 for files
