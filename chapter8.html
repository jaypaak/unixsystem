1. Concepts and Types of Processes

  • Process: A running program.
  • System Process: Required for Unix operations.
  • User Process: Processes initiated by users.
  • Daemon: Background service process managed by the kernel.
  • Parent: Creates child processes.
  • Child: Created by a parent, returns result upon termination.
  • Orphan: Parent process ended before the child.
  • Zombie: Terminated but still holds system resources.

2. Process Management Commands

View Process List

  • ps: Displays process status.
  • pgrep: Searches for processes by name.

Terminate Process

  • kill [signal] PID: Sends a signal to a process.
  • pkill [signal] process_name: Sends a signal to a named process.

Signals

  • 1 (SIGHUP): Terminal disconnect – terminates process.
  • 2 (SIGINT): Ctrl+C – terminates process.
  • 9 (SIGKILL): Forcefully kills process – cannot be ignored.
  • 15 (SIGTERM): Default terminate signal – can be ignored.

Foreground and Background Process Control

  • fg: Bring background job to foreground.
  • bg: Resume job in background.
  • jobs: List background jobs.
  • nohup command &: Run command in background after logout.

Process Monitoring Tools

  • prstat: Real-time process statistics (Solaris).
  • top: Displays live system processes and statistics.

3. User Information Commands

  • users: Displays logged-in users.
  • who: Displays who is logged in and related info.
  • w: Shows who is logged in and what they are doing.
  • who am i: Displays current session user info.
  • whoami: Displays current user's login name.
  • id: Displays user ID, group ID.
  • su: Switch user (default to root if unspecified).
